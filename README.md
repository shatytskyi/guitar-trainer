# Guitar Trainer

Mobile-first PWA that drills guitar chords. Pick a root, see the diagram, hear the chord. Works offline once installed.

**Live:** https://shatytskyi.github.io/guitar-trainer/

UI in Russian, English, Ukrainian. Light + dark themes. International chord notation (`A`, `Am`, `F#m7`, `Cmaj7`).

## Quick start

```bash
npm install
npm run dev      # http://localhost:5173/guitar-trainer/
```

## Scripts

| Command              | What it does                              |
| -------------------- | ----------------------------------------- |
| `npm run dev`        | Vite dev server with hot reload           |
| `npm run build`      | Type-check then build production assets   |
| `npm run preview`    | Serve the production build locally        |
| `npm run test`       | Run unit tests once                       |
| `npm run test:watch` | Run unit tests in watch mode              |
| `npm run typecheck`  | Run TypeScript without emitting           |

## Tech stack

- **Vite** + **TypeScript (strict)** — no UI framework, plain DOM components
- **Tone.js** for audio playback
- **vite-plugin-pwa** (Workbox) for service worker + manifest
- **Vitest** for unit tests
- **GitHub Pages** via `actions/deploy-pages` (typecheck + tests gate every push)

## Design

The Foliage design system — warm-paper palette, forest-green accent, ochre highlights, **Spectral** + **IBM Plex Mono** typography — was generated by [Claude Design](https://www.anthropic.com/news/claude-design-anthropic-labs) and ported into `src/styles/`. Dark mode (`data-theme="stage"`) is part of the same token set.

## Deployment

Push to `main` → workflow runs typecheck → tests → build → publish. A broken `main` never reaches Pages. Forking? Update `base` in `vite.config.ts` and the paths in `index.html` + `public/manifest.webmanifest` to match your repo slug.
